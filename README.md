# Сосед.Велком — сервис аренды вещей у соседей

> Проект по дисциплине «UI/UX-дизайнер». Интерактивный прототип веб-сервиса для P2P-шеринга (аренда вещей между соседями).

**Живой сайт (GitHub Pages):** [Сайт](https://dekoy4.github.io/UI-UX/)

---

## Техническое задание (ТЗ)

### 1. Формальная постановка задачи

**Наименование проекта:** Сайт-сервис «Сосед.Велком»

**Цель разработки:**  
Создать интерактивный прототип веб-сайта, позволяющего пользователям:
- Просматривать доступные для аренды вещи
- Искать товары по ключевым словам
- Бронировать вещи на определённые даты
- Отслеживать свои активные и завершённые аренды

**Целевая аудитория:**  
Жители многоквартирных домов в возрасте 25–45 лет, которым необходимо временно использовать вещи (инструменты, спортивный инвентарь, техника) без покупки.

**Функциональные требования:**

| Модуль | Описание |
|--------|----------|
| Главная страница | Поисковая строка, лента популярных объявлений (4 карточки) |
| Карточка товара | Фото, цена за сутки, информация о владельце (аватар, имя, этаж, рейтинг), описание, правила, календарь для выбора дат, кнопка «Снять» |
| Бронирование | Детали заказа, расчёт суммы (аренда + залог), поле для сообщения владельцу, кнопка отправки запроса |
| Личный кабинет | Профиль пользователя, две вкладки: «Арендую я» (активные/завершённые аренды), «Сдают у меня» (мои объявления, статусы) |

**Технические требования:**
- Адаптивная вёрстка (Desktop 1440px → Mobile 390px)
- Кликабельный прототип с навигацией между страницами
- Визуальное состояние элементов при наведении (hover)
- Локальное хранение изображений (без внешних зависимостей)
- Доступность для людей с ограниченными возможностями (WCAG 2.1)

**Технологический стек:**
- HTML5
- CSS3 (Flexbox, Grid, медиазапросы)
- JavaScript (поиск, динамическое отображение дат, переключение табов)

### 2. Неформальная постановка задачи (в формате брифа)

**Контекст**  
В многоквартирном доме живут люди, у которых есть полезные вещи: дрель, стремянка, проектор, сноуборд. В среднем каждая вещь используется 1–2 раза в год, остальное время пылится в кладовке. При этом сосед сверху или снизу мог бы с радостью арендовать эту вещь на пару дней, но не знает, у кого спросить. Обычные доски объявлений (Avito, Юла) перегружены продажами, мошенниками и не дают ощущения «соседской» безопасности.

**Боли пользователя (арендатора):**
- «Неудобно писать в общий чат дома: "У кого есть дрель?"»
- «Боюсь, что сломаю чужую вещь — непонятно, как будут решаться конфликты»
- «Нет доверия к незнакомому человеку — хочу видеть рейтинг и фото»

**Боли пользователя (сдающего):**
- «Лень фоткать и писать длинное объявление»
- «Боюсь, что вещь не вернут или испортят»
- «Не знаю, сколько брать в сутки — нужна подсказка»

**Задача дизайнера**  
Сделать интерфейс с человеческим лицом. Убрать страх. Добавить доверия: фото соседа, рейтинг, этаж (вместо абстрактных метров). Аренда должна ощущаться как «дать другу на время», а не как бюрократическая сделка.

**Ключевые слова, которыми мы руководствовались:**
- Доверие (аватарки, рейтинги, верификация)
- Простота (минимум полей, кнопка «Снять» на видном месте)
- Соседство (этаж вместо метров, локация «рядом с вами»)
- Доступность (инклюзивный дизайн для всех пользователей)

### 3. Доступность ( Accessibility)

Проект разработан с учётом потребностей людей с ограниченными возможностями здоровья. Реализованы следующие принципы доступности в соответствии с рекомендациями WCAG 2.1 (уровень AA).

#### 3.1. Воспринимаемость (Perceivable)

| Критерий | Реализация |
|----------|------------|
| Альтернативный текст для изображений | Все изображения имеют осмысленные `alt`-атрибуты с описанием содержимого |
| Адаптивная типографика | Относительные единицы измерения, текст масштабируется без потери смысла |
| Контрастность | Соотношение контраста текста и фона не менее 4.5:1 (проверено) |
| Использование цвета | Цвет не является единственным способом передачи информации (дополнительные текстовые метки) |

#### 3.2. Управляемость (Operable)

| Критерий | Реализация |
|----------|------------|
| Клавиатурная навигация | Все интерактивные элементы доступны через Tab, Enter, Space |
| Skip link | Ссылка «Перейти к основному содержанию» для пропуска навигации |
| Фокус | Визуальный индикатор фокуса для клавиатурной навигации (`:focus-visible`) |
| Достаточное время | Нет ограничений по времени для выполнения задач |

#### 3.3. Понятность (Understandable)

| Критерий | Реализация |
|----------|------------|
| Предсказуемая навигация | Единая структура навигации на всех страницах |
| Подписи полей | Все поля форм имеют видимые и скрытые (для скринридеров) подписи |
| Язык страницы | Атрибут `lang="ru"` для корректной работы скринридеров |
| Инструкции | Понятные плейсхолдеры и подсказки в полях ввода |

#### 3.4. Надёжность (Robust)

| Критерий | Реализация |
|----------|------------|
| Семантическая разметка | Использование правильных HTML-тегов (`header`, `nav`, `main`, `section`, `article`) |
| ARIA-атрибуты | Добавлены `role`, `aria-label`, `aria-labelledby`, `aria-current`, `aria-live` |
| Живые регионы | `aria-live="polite"` для объявления результатов поиска |
| Проверка валидности | HTML-код проверен на валидность W3C-валидатором |

#### 3.5. Использованные ARIA-атрибуты

| Атрибут | Применение |
|---------|------------|
| `role="banner"` | Шапка сайта |
| `role="navigation"` | Навигационное меню |
| `role="main"` | Основное содержимое страницы |
| `role="search"` | Блок поиска |
| `role="list"` / `role="listitem"` | Список карточек товаров |
| `role="status"` | Область для объявления результатов поиска |
| `aria-label` | Дополнительные метки для скринридеров |
| `aria-labelledby` | Связь заголовка с секцией |
| `aria-current="page"` | Индикация текущей страницы в навигации |
| `aria-live="polite"` | Объявление результатов поиска без прерывания |
| `aria-hidden="true"` | Скрытие декоративных иконок от скринридеров |

#### 3.6. Как тестировалась доступность

1. **Клавиатурное тестирование** — полная навигация по сайту с помощью Tab, Enter, Space
2. **Скринридеры** — проверка в VoiceOver (macOS) и NVDA (Windows)
3. **Контрастность** — проверка с помощью инструментов разработчика Chrome Lighthouse
4. **Масштабирование** — проверка при увеличении страницы до 200%

## UI/UX решения

### Цветовая палитра

| Роль | Цвет | Код | Контрастность |
|------|------|-----|---------------|
| Акцентный (кнопки, ссылки) | Мягкий зелёный | #4CAF80 | 4.8:1 (AA) |
| Фон страницы | Светло-серый | #F9FAFB | — |
| Карточки, модальные окна | Белый | #FFFFFF | — |
| Основной текст | Тёмно-серый | #1F2937 | 14.2:1 (AAA) |
| Второстепенный текст | Серый | #6B7280 | 7.6:1 (AAA) |

Все цветовые комбинации соответствуют уровню AA по контрастности.

### Типографика
- Основной шрифт: системный (-apple-system, BlinkMacSystemFont, 'Segoe UI')
- Заголовки: от 24px до 48px с градиентом на главном экране
- Тело текста: 16px, межстрочный интервал 1.5
- Относительные единицы для доступного масштабирования

### Ключевые UI-элементы

| Элемент | Решение | Доступность |
|---------|---------|-------------|
| Карточка товара | Скругление 20px, тень, ховер с увеличением | Фокус при Tab, открытие по Enter |
| Кнопка «Снять» | Зелёная, скруглённая, с анимацией нажатия | Доступна с клавиатуры, понятный текст |
| Аватарки | Круглые, реальные фото (не иконки) | Alt-текст с именем владельца |
| Рейтинг | Звёзды + число отзывов | Текстовое описание для скринридеров |
| Поиск | Работает по ключевым словам, показывает счётчик | Роль search, aria-live для результатов |
| Пустое состояние | Иконка коробки + текст + кнопка «На поиски» | ARIA-атрибуты для скринридеров |

### Адаптивность

| Брейкпоинт | Поведение |
|------------|-----------|
| > 768px (Desktop) | Карточки в 3 колонки, двухколоночная раскладка карточки товара |
| < 768px (Mobile) | Карточки в 1 колонку, поиск вертикальный, кнопки на всю ширину |

## Функционал в деталях

### Поиск
- Поиск по ключевым словам (дрель, сноуборд, стремянка, проектор, инструмент, кино)
- Работает по кнопке «Найти» и по клавише Enter
- Показывает количество найденных объявлений
- При пустом поиске — показывает все объявления
- Если ничего не найдено — сообщение о том, что ничего не найдено
- Результаты объявляются скринридером через `aria-live`

### Бронирование
- Универсальная страница, подставляющая данные выбранного товара
- Автоматический расчёт стоимости (цена за сутки × количество дней + залог)
- Поле для сообщения владельцу
- Кнопка отправки запроса с персонализированным сообщением

### Личный кабинет
- Вкладка «Арендую я» — активные и завершённые аренды
- Вкладка «Сдают у меня» — мои объявления и статусы (ожидает / активно)
- Статусы с цветовыми индикаторами и текстовыми метками
- Переключение вкладок доступно с клавиатуры

## Тестирование и проверка

### Сценарии для проверки преподавателем

**1. Главная страница**
- Набрать в поиске «дрель» → должна остаться только дрель
- Набрать «инструмент» → найти дрель и стремянку
- Набрать «абоба» → показать сообщение «ничего не найдено»

**2. Карточка товара**
- Кликнуть на любую карточку → открылась страница с правильным товаром
- Кликнуть на табы «Описание» / «Правила» → переключение контента
- Выбрать даты → пересчиталась цена

**3. Бронирование**
- Нажать «Снять» → открылась страница бронирования
- Данные товара и владельца соответствуют выбранному

**4. Личный кабинет**
- Переключение между вкладками «Арендую я» / «Сдают у меня»
- Во вкладке «Сдают у меня» есть товары Алексея (не чужие)

**5. Доступность**
- Нажать Tab → появляется «Перейти к основному содержанию»
- Продолжить навигацию по ссылкам и кнопкам с клавиатуры
- Нажать Enter на карточке → открывается товар
- Включить скринридер → все элементы читаются осмысленно

**6. Адаптив**
- Уменьшить окно браузера до 375px → карточки в 1 колонку, поиск вертикальный
